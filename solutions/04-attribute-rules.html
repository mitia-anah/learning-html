Understanding HTML Attributes

What are HTML Attributes?

HTML attributes provide additional information about HTML elements. They are always specified in the start tag and usually come in name/value pairs like name="value". Attributes modify the default behavior of an element or provide metadata.

Common Attribute Rules

  1. Always use lowercase for attribute names - Attribute names should be in lowercase (e.g., class="main" not CLASS="main")
  2. Always quote attribute values - Use quotes around all attribute values, even if they don't contain spaces
  3. Use double quotes, not single quotes - While both work, double quotes are the HTML standard (e.g., href="page.html")
  4. No spaces around the = sign - Write id="header" not id = "header"
  5. Boolean attributes don't need values - For attributes like disabled, required, just include the name
  6. Separate multiple attributes with spaces - Example: <input type="text" name="username" required>
  7. Use meaningful values - Choose descriptive names for ids and classes

Examples

Correct Usage:

  • <a href="https://example.com" title="Visit Example">Link</a>
  • <img src="photo.jpg" alt="A beautiful sunset" width="300">
  • <input type="email" name="email" required>
  • <div class="container" id="main-content">

Incorrect Usage:

  • <a HREF=page.html>Link</a> - uppercase and unquoted
  • <img src = 'photo.jpg'> - spaces around equals and single quotes
  • <input type="email" required="required"> - boolean attribute doesn't need value
  • <div class=main content> - unquoted value with space
