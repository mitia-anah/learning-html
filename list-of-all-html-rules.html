What are the rules for writing HTML?

There are many rules for coding or writing HTML, but here are some examples of that.

  1. First thing you need is to define a doctype at the top.
  2. For writing element, you can’t have an end tag without start tag, that’s mean, we always need the form less than/greater than characters because tags are always surrounded by angle brackets.
  3. We should not use uppercase but use lowercase instead when write an element name.
  4. Make sure that there is no open bracket or close bracket inside of the tag.
  5. About nesting rules, nest element properly. For example, you may have open header tag actually there is no header element inside of the header tag. Or maybe it happens like this, you may have an open the header tag but you forgot to close it, and all the rest of your child element become header element. So have an attention when nesting element to avoid these accidents.
  6. Need to be good structure in other word it should have good indenting. As we see before, it is always happened when making an arrangement of child element
  7. To add on what I have been accounted for, following the attribute rules also is one thing that we should not missed.
